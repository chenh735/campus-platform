# 校园课程学习与项目协作平台 - 系统设计

## 1. 系统架构

```text
浏览器
  |
  | HTTP / HTTPS
  v
Nginx
  |-- 静态资源 -> Vue 前端 dist
  |
  | /api/*
  v
Rust Axum 后端
  |-- MySQL：用户、课程、评价、资料、项目、点赞、审核记录
  |
  |-- uploads：学习资料文件
```

系统采用前后端分离架构：

- 前端负责页面展示、表单交互、路由跳转。
- 后端负责接口、鉴权、权限控制、数据读写、文件上传。
- MySQL 保存业务数据。
- 文件保存在服务器本地目录。

### 2.2 页面模块

- Auth：登录、注册、验证码。
- Course：课程列表、课程详情、评价、资料。
- Project：招募列表、招募详情、发布招募、申请加入。
- Profile：个人资料、我的评价、我的资料、我的招募、我的申请。

管理员端：

- Dashboard：统计概览。
- CourseAdmin：课程增删改查。
- ContentAdmin：资料、评价、项目审核。
- UserAdmin：用户查询、禁用、恢复。
- ProjectAdmin：项目招募管理。

### 2.3 API 封装

前端统一在 `src/api` 中封装请求：

```text
api/
  auth.ts
  courses.ts
  reviews.ts
  materials.ts
  projects.ts
  profile.ts
  admin.ts
```

请求规则：

- 所有接口统一以 `/api` 开头。
- 登录后请求头携带 `Authorization: Bearer <token>`。
- 响应错误统一在拦截器中处理。
- 401 跳转登录页。
- 403 显示无权限提示。

## 3. 后端模块设计

```text
backend/src/modules/
  auth/        注册、登录、验证码、JWT
  users/       用户信息、经验值、个人中心
  courses/     课程查询与管理
  reviews/     课程评价、评价点赞
  materials/   学习资料上传、下载、审核、点赞
  projects/    项目招募、申请
  admin/       后台统计、管理操作日志
  upload/      文件保存与校验
```

后端公共模块：

- `config`：读取环境变量。
- `db`：MySQL 连接池。
- `error`：统一错误类型和响应。
- `middleware/auth`：解析 JWT、注入当前用户。
- `middleware/role`：管理员权限校验。

## 4. 权限规则

| 功能 | 游客 | 普通用户 | 管理员 |
| --- | --- | --- | --- |
| 查看课程列表 | 允许 | 允许 | 允许 |
| 查看课程详情 | 允许 | 允许 | 允许 |
| 发表课程评价 | 禁止 | 允许 | 允许 |
| 上传学习资料 | 禁止 | 允许 | 允许 |
| 点赞资料/评价 | 禁止 | 允许 | 允许 |
| 发布项目招募 | 禁止 | 允许 | 允许 |
| 申请加入项目 | 禁止 | 允许 | 允许 |
| 查看个人中心 | 禁止 | 允许 | 允许 |
| 添加课程 | 禁止 | 禁止 | 允许 |

## 7. 关键业务逻辑

### 7.1 注册邮箱限制

后端在发送验证码和注册时都必须检查邮箱后缀：

```text
email.ends_with("@mail2.sysu.edu.cn")
```

前端也做同样校验，但最终以后端校验为准。

### 7.2 课程平均评分

课程平均评分从 `course_reviews` 动态计算：

```sql
SELECT AVG(rating), COUNT(*)
FROM course_reviews
WHERE course_id = ? AND status = 'visible';
```

如果后续访问量变大，可以在 `courses` 表中增加 `rating_avg` 和 `review_count` 缓存字段。

### 7.3 点赞与经验值

点赞资料时需要在事务中完成：

1. 查询资料作者。
2. 如果点赞者是作者本人，只记录点赞或直接禁止点赞，但不增加经验值。
3. 插入点赞记录。
4. 更新资料 `like_count + 1`。
5. 更新作者 `experience + 1`。

取消点赞时同样使用事务：

1. 删除点赞记录。
2. 更新资料 `like_count - 1`。
3. 更新作者 `experience - 1`。
4. 经验值最低不低于 0。

评价点赞逻辑相同。

### 7.4 文件上传

后端校验：

- 用户已登录。
- 文件大小不超过配置限制。
- 文件扩展名在允许范围内。
- 所属课程存在且状态正常。

文件命名建议：

```text
uploads/materials/{year}/{month}/{uuid}_{original_name}
```

数据库保存文件路径和原始文件名。

### 7.5 内容状态

学习资料状态：

- pending：待审核
- approved：已通过
- rejected：已驳回
- hidden：已隐藏

课程评价状态：

- visible：可见
- hidden：管理员隐藏
- deleted：用户删除

项目状态：

- recruiting：招募中
- full：已满员
- closed：已关闭
- hidden：管理员隐藏

### 8.3 Nginx 配置思路

```nginx
server {
    listen 80;
    server_name example.com;

    root /var/www/campus-platform/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 8.4 初始化流程

1. 创建 MySQL 数据库 `campus_platform`。
2. 执行数据库 migration。
3. 创建第一个管理员账号。
4. 创建 uploads 目录并配置写入权限。
5. 构建前端并复制到 Nginx 静态目录。
6. 构建 Rust 后端二进制文件。
7. 使用 systemd 或 Docker Compose 启动后端。
8. 配置 Nginx 并重启。

## 9. 测试场景

### 9.1 账号测试

- 非 `@mail2.sysu.edu.cn` 邮箱不能发送验证码。
- 非 `@mail2.sysu.edu.cn` 邮箱不能注册。
- 正确验证码可以注册。
- 错误或过期验证码不能注册。
- 禁用用户不能登录。

### 9.2 课程测试

- 游客可以查看课程列表和详情。
- 普通用户可以发表评价。
- 同一用户不能重复评价同一课程。
- 编辑评价后课程评价列表更新。
- 管理员隐藏评价后用户端不可见。

### 9.3 资料测试

- 普通用户可以上传允许类型的文件。
- 超过大小限制的文件不能上传。
- 待审核资料普通用户不可下载。
- 审核通过后资料可下载。
- 点赞资料后作者经验值增加。
- 取消点赞后作者经验值减少。

### 9.4 项目招募测试

- 普通用户可以发布招募。
- 游客可以查看招募列表。
- 登录用户可以申请加入。
- 同一用户不能重复申请同一项目。
- 发布者可以在个人中心查看申请。
- 管理员可以隐藏违规招募。

### 9.5 权限测试

- 未登录用户访问个人中心会跳转登录页。
- 普通用户访问 `/admin` 会被拒绝。
- 管理员可以访问后台管理页面。
- 用户不能编辑他人发布的评价、资料或项目。
