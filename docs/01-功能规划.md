# 校园课程学习与项目协作平台 - 功能规划

## 1. 项目定位

本平台面向软件工程学院内部学生和管理员使用，目标是把课程学习经验、学习资料共享、课程评价、项目或比赛组队招募集中到一个系统中。

平台分为三个部分：

- 用户端：学生浏览课程、查看评价、上传资料、点赞、发布项目招募、管理个人内容。
- 管理员端：维护课程信息、审核内容、管理用户和项目招募。
- 后端服务：处理账号鉴权、数据读写、文件上传、权限校验和经验值计算。

## 2. 用户角色

| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| 游客 | 未登录用户 | 查看公开课程和项目列表，不可评价、上传、点赞、发布 |
| 普通用户 | 使用 `@mail2.sysu.edu.cn` 邮箱注册的学生 | 评价课程、上传资料、点赞、发布项目、管理个人内容 |
| 管理员 | 系统初始化或后台创建的管理账号 | 管理课程、审核资料和评价、管理用户、处理违规内容 |

## 3. MVP 功能范围

MVP 优先完成以下功能：

1. 邮箱注册与登录
2. 课程列表与课程详情
3. 课程评分与评价
4. 学习资料上传、查看、下载、点赞
5. 评价点赞
6. 经验值与等级
7. 项目或比赛招募列表、发布与查看
8. 个人中心
9. 管理员课程管理
10. 管理员内容审核

## 4. 账号系统

### 4.1 注册

- 用户输入邮箱、验证码、密码、昵称。
- 邮箱必须以 `@mail2.sysu.edu.cn` 结尾。
- 系统发送邮箱验证码。
- 验证码正确且未过期时允许注册。
- 同一个邮箱只能注册一个账号。

### 4.2 登录

- 使用邮箱和密码登录。
- 登录成功后后端返回 JWT Token。
- 前端保存 Token，并在后续请求中携带。

### 4.3 账号状态

用户账号状态包括：

- 正常：可以使用全部普通用户功能。
- 禁用：不能登录或不能发布内容。
- 待处理：保留状态，用于未来扩展人工审核。

## 5. 课程学习功能

### 5.1 课程列表

用户可以查看课程列表，课程信息包括：

- 课程名称
- 课程代码
- 任课教师
- 学分
- 课程类别
- 平均评分
- 评价数量
- 学习资料数量

支持基础筛选：

- 按课程名称搜索
- 按教师搜索
- 按课程类别筛选
- 按评分排序

### 5.2 课程详情

课程详情展示：

- 课程基本信息
- 课程简介
- 平均评分
- 评价列表
- 学习资料列表
- 上传资料入口
- 发表评价入口

页面布局建议为左侧课程信息，右侧为评价和资料区域。

### 5.3 课程评分与评价

登录用户可以对课程评分并发表评价。

评分范围：

- 1 到 5 分
- 允许半星可以作为后续扩展，MVP 使用整数评分即可

评价内容包括：

- 评分
- 文字评价
- 学习建议
- 难度感受
- 工作量感受

限制规则：

- 同一用户对同一课程只能发布一条主评价。
- 用户可以编辑或删除自己的评价。
- 管理员可以隐藏违规评价。

## 6. 学习资料功能

### 6.1 上传资料

登录用户可以在课程详情页上传学习资料。

资料信息包括：

- 所属课程
- 资料标题
- 资料简介
- 文件
- 标签，例如课件、笔记、作业、复习资料、往年题

文件类型建议支持：

- PDF
- DOCX
- PPT/PPTX
- ZIP
- 图片文件

文件大小由后端配置控制，MVP 建议限制为 50MB。

### 6.2 资料查看与下载

用户可以查看课程下的资料列表，内容包括：

- 标题
- 上传者
- 上传时间
- 文件类型
- 下载次数
- 点赞数
- 审核状态

已通过审核的资料允许下载。

### 6.3 资料点赞

- 登录用户可以点赞他人上传的资料。
- 每个用户对同一资料只能点赞一次。
- 点赞后资料作者经验值 +1。
- 取消点赞后资料作者经验值 -1。
- 用户不能通过点赞自己的资料获得经验值。

## 7. 评价点赞功能

- 登录用户可以点赞他人的课程评价。
- 每个用户对同一评价只能点赞一次。
- 点赞后评价作者经验值 +1。
- 取消点赞后评价作者经验值 -1。
- 用户不能通过点赞自己的评价获得经验值。

## 8. 经验值与等级

经验值用于鼓励用户分享资料和评价。

### 8.1 经验值来源

| 行为 | 经验值变化 |
| --- | --- |
| 他人点赞自己的学习资料 | +1 |
| 他人取消点赞自己的学习资料 | -1 |
| 他人点赞自己的课程评价 | +1 |
| 他人取消点赞自己的课程评价 | -1 |

### 8.2 等级规则

| 等级 | 经验值范围 |
| --- | --- |
| Lv1 | 0 - 9 |
| Lv2 | 10 - 29 |
| Lv3 | 30 - 59 |
| Lv4 | 60 - 99 |
| Lv5 | 100 及以上 |

等级由经验值自动计算，不需要单独手动修改。

## 9. 项目与比赛招募功能

### 9.1 招募列表

用户可以查看项目或比赛招募信息。

展示字段包括：

- 标题
- 类型：课程项目、科研项目、比赛组队、个人项目
- 简介
- 所需人数
- 已有人数
- 技术方向
- 截止时间
- 发布者
- 发布时间
- 状态：招募中、已满员、已关闭

支持筛选：

- 按类型筛选
- 按技术方向搜索
- 只看招募中

### 9.2 发布招募

登录用户可以发布招募信息。

需要填写：

- 标题
- 类型
- 项目简介
- 技术栈或方向
- 需要的队友能力
- 招募人数
- 联系方式
- 截止时间

### 9.3 申请加入

MVP 使用轻量申请机制：

- 用户点击申请加入。
- 填写自我介绍和联系方式。
- 发布者在个人中心查看申请记录。
- 不实现即时聊天。

## 10. 个人中心

个人中心展示：

- 用户头像或默认头像
- 昵称
- 邮箱
- 经验值
- 等级
- 发布的课程评价
- 上传的学习资料
- 发布的项目招募
- 申请过的项目

用户可以：

- 修改昵称
- 修改密码
- 编辑或删除自己的评价
- 编辑或删除自己的资料说明
- 编辑或关闭自己的项目招募

## 11. 管理员端功能

### 11.1 后台首页

展示系统概览：

- 用户数量
- 课程数量
- 资料数量
- 项目招募数量
- 待审核内容数量
- 最近管理员操作记录

### 11.2 课程管理

管理员可以：

- 添加课程
- 编辑课程
- 下架课程
- 搜索课程
- 查看课程评价和资料数量

课程字段包括：

- 课程代码
- 课程名称
- 任课教师
- 学分
- 课程类别
- 课程简介
- 开课学期

### 11.3 内容审核

管理员可以审核：

- 学习资料
- 课程评价
- 项目招募

审核操作包括：

- 通过
- 驳回
- 隐藏
- 删除

MVP 建议学习资料默认需要审核，评价和项目招募默认发布后可见，但管理员可隐藏。

### 11.4 用户管理

管理员可以：

- 查看用户列表
- 搜索用户
- 查看用户经验值和等级
- 禁用用户
- 恢复用户

### 11.5 项目管理

管理员可以：

- 查看所有项目招募
- 隐藏违规招募
- 关闭过期或违规招募

## 12. 通知功能

MVP 可以先做站内基础通知，也可以作为二期功能。

通知场景：

- 资料审核通过或驳回
- 项目申请被提交
- 内容被管理员处理

如果工期有限，通知功能可以先不实现，只在个人中心展示状态。

## 13. 功能优先级

| 优先级 | 功能 |
| --- | --- |
| P0 | 注册登录、课程列表、课程详情、评分评价、资料上传、项目招募、个人中心、管理员课程管理 |
| P1 | 点赞、经验值、资料审核、内容隐藏、用户管理 |
| P2 | 项目申请、站内通知、复杂筛选、统计看板 |
| P3 | 即时聊天、全文搜索、移动端深度优化、MinIO 文件存储 |
